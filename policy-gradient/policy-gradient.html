Day 10：策略梯度

Policy Gradient — 从价值函数到直接优化策略 On-Policy

1. 从 Value-Based 到 Policy-Based

DQN 系列的核心思路：先学 $Q^*(s,a)$，再提取策略 $\pi(s) = \arg\max_a Q(s,a)$。这叫 Value-Based 方法——间接地通过价值函数找策略。

Value-Based 的局限

局限说明例子
连续动作空间$\arg\max_a Q(s,a)$ 无法遍历机器人关节角度 $\in \mathbb{R}^n$
随机策略Q 值只能给确定性策略石头剪刀布需要随机出拳
策略退化$\arg\max$ 在 Q 值接近时不稳定两个动作 Q 值几乎相等
Policy-Based 核心思想

直接参数化策略 $\pi_\theta(a|s)$，通过梯度上升最大化期望回报。

$\underbrace{Q^*(s,a) \to \pi^*(s)}_{\text{Value-Based: 间接}} \quad \longrightarrow \quad \underbrace{\pi_\theta(a|s) \to \text{最大化 } J(\theta)}_{\text{Policy-Based: 直接}}$

2. 策略参数化

离散动作：Softmax 策略

Softmax 策略
$\pi_\theta(a|s) = \frac{\exp(h_\theta(s,a))}{\sum_{a'} \exp(h_\theta(s,a'))}$

偏好值 $h_\theta(s,a)$ 越高的动作，被选中的概率越大，但不会 100%（保持探索）。

连续动作：高斯策略

高斯策略
$\pi_\theta(a|s) = \frac{1}{\sigma_\theta(s)\sqrt{2\pi}} \exp\left(-\frac{(a - \mu_\theta(s))^2}{2\sigma_\theta(s)^2}\right)$

$\mu_\theta(s)$ 是"最佳动作"，$\sigma_\theta(s)$ 控制探索幅度。连续动作空间无需 $\arg\max$！

3. 策略梯度定理

目标函数

策略优化目标
$J(\theta) = \mathbb{E}_{\tau \sim \pi_\theta}[R(\tau)] = \mathbb{E}_{\pi_\theta}\left[\sum_{t=0}^{\infty} \gamma^t R_{t+1}\right]$

推导：对数导数技巧

1展开期望：$J(\theta) = \sum_\tau P(\tau|\theta) R(\tau)$
2对 $\theta$ 求导：$\nabla_\theta J = \sum_\tau \nabla_\theta P(\tau|\theta) R(\tau)$
3利用 $\nabla P = P \cdot \nabla \log P$：$\nabla_\theta J = \mathbb{E}_{\pi_\theta}[\nabla_\theta \log P(\tau|\theta) \cdot R(\tau)]$
4展开 $\log P(\tau|\theta)$：只有 $\pi_\theta$ 依赖 $\theta$，转移概率 $P(s'|s,a)$ 消失！
5最终结果：
策略梯度定理
$\nabla_\theta J(\theta) = \mathbb{E}_{\pi_\theta}\left[\sum_{t=0}^{T-1} \nabla_\theta \log \pi_\theta(A_t|S_t) \cdot G_t\right]$
关键洞察

转移概率 $P(S_{t+1}|S_t,A_t)$ 不依赖 $\theta$，求导后消失！这意味着策略梯度不需要知道环境模型。

直觉：回报为正 → 增大该轨迹上所有动作的概率；回报为负 → 减小概率。

4. REINFORCE 算法

采样轨迹
→
计算 $G_t$
→
计算梯度
→
更新 $\theta$

更新规则

REINFORCE 梯度更新
$\theta \leftarrow \theta + \alpha \sum_t \nabla_\theta \log \pi_\theta(A_t|S_t) \cdot G_t$

手算示例：2 动作 Softmax

初始 $\theta_1 = \theta_2 = 0$，$\pi(\text{左}) = \pi(\text{右}) = 0.5$。Agent 选"右"，奖励 $R=+3$。

$\nabla_{\theta_2} \log \pi(\text{右}|s) = 1 - \pi(\text{右}) = 0.5$，$\nabla_{\theta_1} \log \pi(\text{右}|s) = -0.5$

更新（$\alpha=0.1$）：$\theta_2 \leftarrow 0 + 0.1 \times 0.5 \times 3 = 0.15$，$\theta_1 \leftarrow 0 + 0.1 \times (-0.5) \times 3 = -0.15$

结果：$\theta_2 > \theta_1$ → "右"被选中的概率增大。

5. REINFORCE with Baseline

高方差问题

所有回报为正时，坏动作的概率也在增加（只是增加得少）。减去 baseline 使好的动作得正信号，坏动作得负信号：

带 Baseline 的策略梯度
$\nabla_\theta J(\theta) = \mathbb{E}_{\pi_\theta}\left[\sum_t \nabla_\theta \log \pi_\theta(A_t|S_t) \cdot (G_t - V(S_t))\right]$

为什么 Baseline 不引入偏差？

$\mathbb{E}_{\pi_\theta}[\nabla_\theta \log \pi_\theta(a|s)] = \sum_a \pi_\theta(a|s) \nabla_\theta \log \pi_\theta(a|s) = \nabla_\theta \sum_a \pi_\theta(a|s) = \nabla_\theta 1 = 0$

因此 $\mathbb{E}[\nabla \log \pi \cdot b(s)] = b(s) \cdot 0 = 0$，无偏但降低方差。

优势函数视角

优势 = 实际 - 预期
$A(s,a) = G_t - V(S_t)$

衡量"动作 $a$ 比平均好多少"。正优势 → 增大概率；负优势 → 减小概率。

6. 方差问题与降方差技巧

技巧方法原理
Baseline减去 $V(S_t)$让权重有正有负
优势估计$A_t = G_t - V(S_t)$只强化比预期好的动作
多条轨迹增大 batch size平均降低方差
奖励归一化标准化 $G_t$稳定梯度量级
方差-偏差权衡

MC 回报 $G_t$（高方差、无偏）← $n$-step → TD 目标 $R + \gamma V(S')$（低方差、有偏差）

REINFORCE 用 $G_t$（MC），Day 11 的 Actor-Critic 用 TD 误差——这是 Day 5 TD vs MC 在策略梯度中的体现。

8. 总结

1
策略参数化
直接用 $\pi_\theta(a|s)$ 表示策略，Softmax/高斯
2
对数导数技巧
$\nabla P = P \cdot \nabla \log P$，乘法变加法
3
策略梯度定理
$\nabla J = \mathbb{E}[\nabla \log \pi \cdot G_t]$，不需要模型
4
REINFORCE
MC 采样 + 梯度上升，简单但高方差
5
Baseline
减去 $V(s)$ 无偏降方差
6
优势函数
$A(s,a) = G_t - V(s)$，动作比平均好多少
