Day 3：动态规划

Dynamic Programming — 从贝尔曼方程到可执行算法

1. 核心思想：迭代逼近不动点

贝尔曼方程是一个自我引用的方程组——$V(s)$ 依赖于 $V(s')$，而 $V(s')$ 又可能依赖于 $V(s)$。

贝尔曼期望算子
$\mathcal{T}^\pi(V)(s) = \sum_a \pi(a|s) \sum_{s'} P(s'|s,a) [R + \gamma V(s')]$

我们要找的是 $\mathcal{T}^\pi$ 的不动点：$V^\pi = \mathcal{T}^\pi(V^\pi)$

💡 压缩映射 + Banach 不动点定理

$\mathcal{T}^\pi$ 是 $\gamma$-压缩映射：$\|\mathcal{T}^\pi(V_1) - \mathcal{T}^\pi(V_2)\|_\infty \leq \gamma \|V_1 - V_2\|_\infty$
不管初始值怎么猜，反复应用 $\mathcal{T}^\pi$ 一定收敛到唯一不动点 $V^\pi$。

2. 策略评估（Policy Evaluation）

问题

给定策略 $\pi$，计算 $V^\pi$。

更新公式

迭代策略评估
$V_{k+1}(s) = \sum_{a} \pi(a \mid s) \sum_{s'} P(s' \mid s, a) \Big[ R(s, a, s') + \gamma V_k(s') \Big]$

公式拆解

符号含义来源
$\pi(a|s)$策略：在 $s$ 选 $a$ 的概率已知
$P(s'|s,a)$转移概率已知
$R(s,a,s')$即时奖励已知
$\gamma V_k(s')$对未来价值的折现上一轮结果
整体$V_{k+1}(s)$ = 所有可能路径的期望回报对动作×下一状态加权

流程

V=0 初始化→贝尔曼期望更新→Δ<θ ?→输出 Vπ

手算示例：1×3 网格

[ S ]──[ A ]──[ G ]    γ=0.9, 每步 R=-1, 到 G 奖励 +10
策略: π(S)=→, π(A)=→ (一直向右)

k=0:  V(S)=0,  V(A)=0,  V(G)=0
k=1:  V(S)=-1+0.9×0=-1,  V(A)=10+0.9×0=10,  V(G)=0   ← A 发现 G
k=2:  V(S)=-1+0.9×10=8,  V(A)=10,           V(G)=0   ← S 发现 A
k=3:  收敛! (值不再变化)
信息从终点向起点"倒流"——每轮迭代，价值信息向前传播一步。

3. 策略改进（Policy Improvement）

核心：贪心选 Q 最大的动作

贪心改进
$\pi'(s) = \arg\max_a Q^\pi(s, a) = \arg\max_a \sum_{s'} P(s'|s,a) [R + \gamma V^\pi(s')]$

策略改进定理

🎯 贪心改进一定不会变差

如果 $\pi'$ 由 $\pi$ 经贪心改进得到，则 $V^{\pi'}(s) \geq V^\pi(s)$ 对所有 $s$ 成立。

证明（四步）

1贪心定义：$V^\pi(s) \leq Q^\pi(s, \pi'(s))$，因为 $\pi'$ 选了 Q 最大的动作
2展开 Q：$V^\pi(s) \leq \sum_{s'} P(s'|s,\pi'(s)) [R + \gamma V^\pi(s')]$
3递归应用：对 $s'$ 再次用 $V^\pi(s') \leq Q^\pi(s', \pi'(s'))$，逐层展开
4极限：展开到终止状态，右侧 = $\mathbb{E}_{\pi'}[G_t|S_t=s] = V^{\pi'}(s)$，证毕 $\square$

4. 策略迭代（Policy Iteration）

π₀→评估→Vπ₀→改进→π₁→评估→...→π*
算法结构

repeat: { 策略评估（迭代到 Δ<θ）→ 策略改进（贪心选动作）} until 策略稳定

1×3 网格手算

轮次策略V(S)V(A)说明
初始50%←/50%→3.184.55随机策略价值低
改进后全→810贪心后价值飙升
再改进全→ (不变)810策略稳定，结束

5. 价值迭代（Value Iteration）

动机

策略迭代的评估步很慢。能不能边评估边改进？

价值迭代更新
$V_{k+1}(s) = \max_a \sum_{s'} P(s'|s,a) [R + \gamma V_k(s')]$

和策略评估的唯一区别

策略评估价值迭代
更新公式$\sum_a \pi(a|s) Q(s,a)$$\max_a Q(s,a)$
含义加权平均取最大值
效果只评估，不改进评估 + 隐式改进

收敛证明：压缩映射

T* 是 γ-压缩映射
$$\begin{aligned} &\|T^*(V_1) - T^*(V_2)\|_\infty \\ &\leq \max_s \max_a \sum_{s'} P(s'|s,a) \gamma |V_1(s') - V_2(s')| \\ &\leq \gamma \|V_1 - V_2\|_\infty \underbrace{\sum_{s'} P(s'|s,a)}_{=1} \\ &= \gamma \|V_1 - V_2\|_\infty \end{aligned}$$

由 Banach 不动点定理：$V_k \to V^*$（唯一不动点）。

1×3 网格手算

k=0:  V(S)=0,    V(A)=0,    V(G)=0
k=1:  V(S)=max(-1,-1)=-1,  V(A)=max(-1,10)=10,  V(G)=0
k=2:  V(S)=max(-1,-1+9)=8, V(A)=10,             V(G)=0
k=3:  收敛! 提取策略: π(S)=→, π(A)=→

6. 广义策略迭代（GPI）

策略 π→评估 E→价值 V/Q→改进 I→策略 π
🔑 GPI 是所有 RL 算法的统一框架

评估和改进不必"尽善尽美"——只要两者轮流推进，系统整体向最优策略收敛。

算法评估（E）改进（I）
策略迭代迭代到收敛完全贪心
价值迭代只做一步max（隐式贪心）
